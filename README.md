<h2><a id="user-content-how-it-works" class="anchor" href="#how-it-works" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>How it works</h2>

<p>The Fuck tries to match a rule for the previous command, creates a new command
using the matched rule and runs it. Rules enabled by default are as follows:</p>

<ul>
<li><code>ag_literal</code> â adds <code>-Q</code> to <code>ag</code> when suggested;</li>
<li><code>aws_cli</code> â fixes misspelled commands like <code>aws dynamdb scan</code>;</li>
<li><code>cargo</code> â runs <code>cargo build</code> instead of <code>cargo</code>;</li>
<li><code>cargo_no_command</code> â fixes wrongs commands like <code>cargo buid</code>;</li>
<li><code>cd_correction</code> â spellchecks and correct failed cd commands;</li>
<li><code>cd_mkdir</code> â creates directories before cd'ing into them;</li>
<li><code>cd_parent</code> â changes <code>cd..</code> to <code>cd ..</code>;</li>
<li><code>chmod_x</code> â add execution bit;</li>
<li><code>composer_not_command</code> â fixes composer command name;</li>

<p>For adding your own rule you should create <code>your-rule-name.py</code>
in <code>~/.config/thefuck/rules</code>. The rule should contain two functions:</p>

<div class="highlight highlight-source-python"><pre>match(command: Command) <span class="pl-ii">-&gt;</span> <span class="pl-c1">bool</span>
get_new_command(command: Command) <span class="pl-ii">-&gt;</span> <span class="pl-c1">str</span> <span class="pl-k">|</span> list[<span class="pl-c1">str</span>]</pre></div>

<p>Also the rule can contain an optional function</p>

<div class="highlight highlight-source-python"><pre>side_effect(old_command: Command, fixed_command: <span class="pl-c1">str</span>) <span class="pl-ii">-&gt;</span> <span class="pl-c1">None</span></pre></div>

<p>and optional <code>enabled_by_default</code>, <code>requires_output</code> and <code>priority</code> variables.</p>

<p><code>Command</code> has three attributes: <code>script</code>, <code>stdout</code>, <code>stderr</code> and <code>script_parts</code>.
Rule shouldn't change <code>Command</code>.</p>

<p><em>Rules api changed in 3.0:</em> For accessing settings in rule you need to import it with <code>from thefuck.conf import settings</code>.
<code>settings</code> is a special object filled with <code>~/.config/thefuck/settings.py</code> and values from env (<a href="#settings">see more below</a>).</p>

<p>Simple example of the rule for running script with <code>sudo</code>:</p>

<div class="highlight highlight-source-python"><pre><span class="pl-k">def</span> <span class="pl-en">match</span>(<span class="pl-smi">command</span>):
    <span class="pl-k">return</span> (<span class="pl-s"><span class="pl-pds">'</span>permission denied<span class="pl-pds">'</span></span> <span class="pl-k">in</span> command.stderr.lower()
            <span class="pl-k">or</span> <span class="pl-s"><span class="pl-pds">'</span>EACCES<span class="pl-pds">'</span></span> <span class="pl-k">in</span> command.stderr)


<span class="pl-k">def</span> <span class="pl-en">get_new_command</span>(<span class="pl-smi">command</span>):
    <span class="pl-k">return</span> <span class="pl-s"><span class="pl-pds">'</span>sudo <span class="pl-c1">{}</span><span class="pl-pds">'</span></span>.format(command.script)

<span class="pl-c"># Optional:</span>
enabled_by_default <span class="pl-k">=</span> <span class="pl-c1">True</span>

<span class="pl-k">def</span> <span class="pl-en">side_effect</span>(<span class="pl-smi">command</span>, <span class="pl-smi">fixed_command</span>):
    subprocess.call(<span class="pl-s"><span class="pl-pds">'</span>chmod 777 .<span class="pl-pds">'</span></span>, <span class="pl-v">shell</span><span class="pl-k">=</span><span class="pl-c1">True</span>)

priority <span class="pl-k">=</span> <span class="pl-c1">1000</span>  <span class="pl-c"># Lower first, default is 1000</span>

requires_output <span class="pl-k">=</span> <span class="pl-c1">True</span></pre></div>

<p>The Fuck has a few settings parameters which can be changed in <code>$XDG_CONFIG_HOME/thefuck/settings.py</code> (<code>$XDG_CONFIG_HOME</code> defaults to <code>~/.config</code>):</p>

<ul>
<li><code>rules</code> â list of enabled rules, by default <code>thefuck.conf.DEFAULT_RULES</code>;</li>
<li><code>exclude_rules</code> â list of disabled rules, by default <code>[]</code>;</li>
<li><code>require_confirmation</code> â requires confirmation before running new command, by default <code>True</code>;</li>
<li><code>wait_command</code> â max amount of time in seconds for getting previous command output;</li>
<li><code>no_colors</code> â disable colored output;</li>
<li><code>priority</code> â dict with rules priorities, rule with lower <code>priority</code> will be matched first;</li>
<li><code>debug</code> â enables debug output, by default <code>False</code>;</li>
<li><code>history_limit</code> â numeric value of how many history commands will be scanned, like <code>2000</code>;</li>
<li><code>alter_history</code> â push fixed command to history, by default <code>True</code>;</li>
<li><code>wait_slow_command</code> â max amount of time in seconds for getting previous command output if it in <code>slow_commands</code> list;</li>
<li><code>slow_commands</code> â list of slow commands.</li>
</ul>

<p>Example of <code>settings.py</code>:</p>

<div class="highlight highlight-source-python"><pre>rules <span class="pl-k">=</span> [<span class="pl-s"><span class="pl-pds">'</span>sudo<span class="pl-pds">'</span></span>, <span class="pl-s"><span class="pl-pds">'</span>no_command<span class="pl-pds">'</span></span>]

<p>Or via environment variables:</p>

<ul>
<li><code>THEFUCK_RULES</code> â list of enabled rules, like <code>DEFAULT_RULES:rm_root</code> or <code>sudo:no_command</code>;</li>
<li><code>THEFUCK_EXCLUDE_RULES</code> â list of disabled rules, like <code>git_pull:git_push</code>;</li>
<li><code>THEFUCK_REQUIRE_CONFIRMATION</code> â require confirmation before running new command, <code>true/false</code>;</li>
<li><code>THEFUCK_WAIT_COMMAND</code> â max amount of time in seconds for getting previous command output;</li>
<li><code>THEFUCK_NO_COLORS</code> â disable colored output, <code>true/false</code>;</li>
<li><code>THEFUCK_PRIORITY</code> â priority of the rules, like <code>no_command=9999:apt_get=100</code>,

<li><code>THEFUCK_DEBUG</code> â enables debug output, <code>true/false</code>;</li>
<li><code>THEFUCK_HISTORY_LIMIT</code> â how many history commands will be scanned, like <code>2000</code>;</li>
<li><code>THEFUCK_ALTER_HISTORY</code> â push fixed command to history <code>true/false</code>;</li>
<li><code>THEFUCK_WAIT_SLOW_COMMAND</code> â max amount of time in seconds for getting previous command output if it in <code>slow_commands</code> list;</li>
<li><code>THEFUCK_SLOW_COMMANDS</code> â list of slow commands, like <code>lein:gradle</code>.</li>
</ul>
